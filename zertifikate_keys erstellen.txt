
1. Selbstsigniertes Zertifikat erstellen (einfach)
Dies ist ideal für Entwicklungsumgebungen oder interne Server, da keine offizielle Zertifizierungsstelle (CA) benötigt wird.

    Befehl:

openssl req -x509 -newkey rsa:4096 -keyout mein.key -out mein.crt -days 365 -nodes

Erklärung:

    req -x509: Erstellt ein selbstsigniertes Zertifikat.
    -newkey rsa:4096: Erzeugt einen neuen 4096-Bit RSA-Schlüssel.
    -keyout: Dateiname des privaten Schlüssels.
    -out: Dateiname des Zertifikats.
    -days 365: Gültigkeitsdauer.
    -nodes: (No DES) Erstellt den Schlüssel ohne Passwortschutz (wichtig für Webserver wie Apache/Nginx, damit sie beim Start nicht nach einem Passwort fragen).

2. Privater Key und CSR für offizielle Zertifikate
Um ein vertrauenswürdiges Zertifikat (z.B. bei CA) zu erhalten, wird ein privater Key und ein CSR erstellt.

    Privaten Key erstellen:

openssl genrsa -out server.key 4096

CSR erstellen (Zertifikatsanforderung):

openssl req -new -key server.key -out server.csr

Während der Erstellung werden Daten wie "Common Name" (Domainname) abgefragt.

3. Eigenes CA-Zertifikat erstellen (für interne Zwecke)
Falls Sie eine eigene Zertifizierungsstelle aufbauen wollen:

    CA-Schlüssel erzeugen: openssl genrsa -aes256 -out ca-key.pem 4096
    CA-Zertifikat erzeugen: openssl req -x509 -new -extensions v3_ca -key ca-key.pem -out ca-cert.pem -days 3650

Wichtige Hinweise

    Schlüsselrechte: Der private Key sollte nur für den Root-User lesbar sein: chmod 600 mein.key.
    Speicherorte: Unter Ubuntu/Debian werden Zertifikate oft unter /etc/ssl/certs/ und Keys unter /etc/ssl/private/ abgelegt.
    Let's Encrypt: Für kostenlose, vertrauenswürdige Zertifikate auf öffentlichen Servern wird {Link: Certbot https://certbot.eff.org/} empfohlen.
